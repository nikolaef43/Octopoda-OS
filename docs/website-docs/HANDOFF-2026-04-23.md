## Critical open bugs (known, not fixed)

1. **`/v1/agents/{id}/memory` list endpoint times out on big tenants** (>100k memories per tenant). My own tenant (1f3442be42cfd12f) has 1.16M memories and consistently times out. Affects Memory Explorer dashboard tab + `log_decision`'s cloud memory snapshot capture. Needs Postgres index + GROUP BY query rewrite. ~2-3hr server fix.

2. **Mac: zero tests.** Every doc says "should work on Mac" but has never been verified. Pure Python SDK suggests identical behaviour to Linux. Apple Silicon + `octopoda[ai]` extras (torch) could fail unpredictably. Needs a Mac to test.

3. **OpenClaw integration: zero tests.** Documented in MCP-adjacent section but never wired end-to-end due to Node.js toolchain not being available in test env.

4. **LangChain docs on live website currently show `octopoda.LangChainMemory`** — that class is broken on LangChain 1.x (pydantic ValidationError the moment it's wrapped in RunnableWithMessageHistory). Any current LangChain user following the live docs hits an immediate crash. **This is likely a primary cause of the 4% real activation rate.** Fix = deploy the new 03-langchain.md.

5. **Agent cap hit on owner tenant during testing** — temporarily set max_agents to INT_MAX on tenant 1f3442be42cfd12f directly in DB to continue testing. Worth reviewing before FoundersEdge demos.
